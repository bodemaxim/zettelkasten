# Задание на рефакторинг

## Вынос констант

Мы не объявляем константы (литералы типа `number` или объекты) во .vue файле. Если внутри *.vue объявлен константа (литерал типа `number` или объект), его надо перенести в файл *.consts.ts в той же папке. Если в *.vue есть объявление констант, а файл *.consts.ts отсутствует - создай этот файл и перенеси в него константу, используя `export const`.

Например, есть файл `edit-device-panel.vue` и в нем объявлена константа. Тогда надо перенести объявление константы в `edit-device-panel.consts.ts` в той же папке.

Литеральные константы в SCREAMING_SNAKE_CASE. Константы-объекты в camelCase.

ВАЖНО! Не вноси никаких изменений кроме указанных в этом файле. НЕ НАДО переносить типы (`type`) в .consts!
ВАЖНО! Если файлы соответствуют описанным правилам, не вноси изменений - это нормально.

## Порядок констант
ВАЖНО! Даже если все константы уже добавлены файл, их еще нужно ОТСОРТИРОВАТЬ.

Отсортируй константы по следующему принципу. Ниже даны три блока. Между каждым блоком добавляй одну пустую строку для визуального отделения.

1. Литеральные константы в SCREAMING_SNAKE_CASE. В этом блоке без "blank line" между строками.
    ```
    export const MAX_ZOOM = 22;
    ```

2. Объекты, значение которых присвоено в одну строчку: из словаря, объекта-константы, функции-однострочника.
    ```
    export const deviceStatusesEnum = getEnumValues(enumConfig.Device.status);
    ```

3. Объекты, значение которых присвоено из колбэка или IIFE.
    ```
    export const deviceStatusOptionsList: LabelledSelectOption[] = enumConfig.Device.status.map(
    ({ value, label }) => ({ value, label }),
    );
    ```

4. Объекты, значение которых присвоено напрямую. В этом блоке НУЖНЫ "blank line" между строками.
    ```
    export const defaultDevice: NormalizeStrings<DeviceWritable> = {
    status: deviceStatusesEnum.ENABLED,
    inventory_id: '',
    trilateration_calibration_enabled: false,
    };
    ```
